Introduction

Git and GitHub are the backbone of modern software engineering. While Git handles versioning locally, GitHub provides the cloud infrastructure for collaboration.

What is Git?

Git is a distributed version control system (DVCS). Unlike older systems, every developer's computer has a full copy of the project history, making it fast and reliable.

What is GitHub?

GitHub is a cloud-based platform that provides hosting for Git repositories, along with collaboration and project management features.

Git Installation

Terminal
# Ubuntu / Linux
sudo apt install git

# Windows

Download Git from
https://git-scm.com
# macOS
brew install git

Basic Git Commands

Terminal
git init                      // Start a new repo
git status                    // Check file states
git add .                     // Stage all changes
git commit -m "feat: login"   // Save snapshot
git log --oneline             // View history

Git Workflow

Git operates across three main "areas": the Working Directory, the Staging Area, and the Repository.

Working with GitHub

Terminal

git remote add origin 
git push -u origin main

Branching & Merging

Terminal

git branch feature-branch
git checkout feature-branch
git merge feature-branch

Best Practices

  • Atomic Commits: Keep changes small and focused.
  • Descriptive Messages: Use the imperative mood (e.g., "Fix bug" not "Fixed bug").
  • Pull Frequently: Avoid merge conflicts by staying up to date.
